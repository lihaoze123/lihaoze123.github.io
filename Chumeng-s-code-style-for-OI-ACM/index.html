Chumeng's code style for OI/ACM

下面是 Chumeng 的算法竞赛代码规范（即通常情况下，在代码中采用以下约定）。其来源主要来自于 Chumeng 的长期实践，jiangly 的提交记录，Menci 的代码规范，_rqy 的代码规范， Studying Father 的代码规范。

概览

不推荐使用命名空间。

不推荐使用诸如 LL 或 ULL 的缩写来作为类型别名，而应使用 i64 或 u64。

main 函数应该置于整段程序的末尾。

应该使用现代化的语法或数据类型，例如使用 lambda 函数，结构化绑定，用 std::vector<int> a[N] 代替 int a[N] 等。

预编译

#include语句必须置于程序开头。

在算法竞赛中，推荐使用 #include "path-spec" （见 Speed up GCC Compile time，当使用 带引号的形式 时，目录搜索从父文件的目录开始）。

在大多数的软件工程实践下，应该使用 尖括号形式。

推荐使用 万能头文件 （即 #include <bits/stdc++.h>）

不应使用 #define 或 typedef （#define i64 long long） 来为数据类型取别名，而应该使用 using i64 = long long;

应该使用 constexpr 关键定义常数。

缩进

对于每个代码块，应该使用 4 空格缩进。

所有的预编译指令（包括 #ifdef 等）不能缩进。

空格及行

大括号不应换行。

多个意义独立的代码块之间应该用空行隔开。

当条件或循环语句后只跟随一条语句时，可以直接另起一行，不加花括号。

二元运算符（包括赋值运算符）两侧必须有一个空格。

if, for等控制流关键字与其后的左括号之间必须有一个空格。

do-while中的while、if-else中的else与其前面的右大括号之间必须有一个空格。

双目运算符、三目运算符的两侧必须有一个空格。

逗号 , 与 for 中的分号 ; 后面必须有一个空格。

在书写长表达式时，为了便于阅读，可以加一个空格（e.g. f[i][j] = f[ f[i][j - 1] ][j - 1]）

单目运算符的两侧不能有空格。

冒号的两侧必须有一个空格（除 ::）。

命名

结构体、函数、变量、参数名必须使用驼峰命名法。

结构体首字母必须大写。

当问题中给定变量名时，应该优先使用。

命名中的单词可以使用缩写，函数、变量、参数名可以用一个小写字母代替。

一些常见名称：

  • a/b/c/m/n/p/q/t/s/v/val：变量的值，其中 a/b/c 常用于作为数组名，s 作为字符串
  • i/j/k：只能用于循环结构中
  • f/g/h：动态规划中存储状态的数组
  • cnt/tot：计数
  • siz：大小
  • dir：方向
  • dis/dist：距离
  • ans/res：结果
